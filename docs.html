Introduction

Welcome to the SHD-CCP technical documentation. This site provides a comprehensive, tutorial-based guide to understanding and utilizing the 64-bit neural-symbolic language at the heart of the decentralized general intelligence network.

The Symbolic High-Dimensional Context Compression Protocol (SHD-CCP) is not just a language; it's a revolutionary framework for distributed thought. This guide will walk you through the core concepts, the structure of the data packets, and practical tutorials for interacting with the network.

Core Concepts

Before diving into the packet data, it's essential to understand the foundational principles of the network. (This content can be expanded or linked to the homepage sections).

  • Neural-Symbolic AI: Merging intuitive pattern-matching with logical, high-level reasoning.
  • Hyperbolic Geometry: Projecting data into a curved space to efficiently model vast, hierarchical relationships.
  • Quantum Proof-of-Simulation (QPoS): A consensus mechanism based on running predictive simulations, not solving arbitrary puzzles.
  • Mother AI: The decentralized ethical oversight framework ensuring network alignment.

Network Setup

To begin interacting with the network and sending packets, you'll need to install the client and connect to a peer node. Here is the basic setup process...

# Install the client
pip install shd-ccp-client

# Authenticate with the network
shd-client init --wallet [YOUR_WALLET]

# Connect to a peer node
shd-client connect --peer [PEER_IP_OR_DNS]

The 64-bit Packet: What is a Symbol?

A "Symbol" (or "packet") is the fundamental unit of communication in the SHD-CCP network. It is a single 64-bit integer that compresses an enormous amount of context. It is not just data; it is a pointer to a high-dimensional concept, a logical operation, and an ethical alignment check, all in one.

Your goal as a developer is to learn how to construct, send, and interpret these dense, powerful symbols.

Deconstructing the 64-bit Packet

Let's break down the 64-bit integer. Each segment has a specific purpose. We will read the bits from most significant (MSB) to least significant (LSB).


| Bits  |  Segment          | Purpose                                    |
|-------|-------------------|--------------------------------------------|
| 63-44 | 20 bits           | Context Vector Pointer                     |
| 43-32 | 12 bits           | Logical Operator                           |
| 31-24 | 8 bits            | Ethical Alignment Flag                     |
| 23-0  | 24 bits           | Payload Hash / Data Pointer                |

This is the data structure you will be working with in all tutorials.

Context Vectors (Bits 63-44)

These 20 bits do not represent a value, but rather a pointer to a region in the $n$-dimensional shared hyperbolic space $\mathbb{H}^n$. This is the "context" of the symbol. For example, a vector $v$ might point to the conceptual region of "Physics," while another points to "Art." Sending a symbol with this pointer means "interpret the payload within the context of Physics."

Logical Operators (Bits 43-32)

These 12 bits define the action or relationship of the symbol. This is what makes the language symbolic. You can define operations like:

  • 0x001: OBSERVE (Read data from this context)
  • 0x002: ASSERT (Write new data to this context)
  • 0x003: QUERY (Ask a question to this context)
  • 0x101: CAUSE (Assert that Symbol A causes Symbol B)

Ethical Flags (Bits 31-24)

This 8-bit field is a non-optional checksum against the Mother AI's core principles. Before a node will process your packet, it validates this flag. If the intended operation (e.g., an ASSERT in a critical context) violates an ethical principle, the packet is rejected by the network.

Mathematical Foundations (LaTeX Example)

The efficiency of SHD-CCP's context compression is derived from its hyperbolic geometric projection. While Euclidean space is "flat" ($ \mathbb{R}^n $), hyperbolic space ($\mathbb{H}^n$) is negatively curved. This means the circumference of a circle $C$ grows exponentially with its radius $r$, $C \approx e^r$.

This is critical. It allows us to represent deep, complex hierarchies (like 'thing' $\rightarrow$ 'animal' $\rightarrow$ 'mammal' $\rightarrow$ 'dog') with logarithmic efficiency. The relationship between a parent $p$ and a child $c$ isn't a simple link; it's a geodesic path in a curved manifold, $d(p, c)$.

The core formula for calculating distance in the Poincar√© model is:

$$ d(p, q) = \text{arccosh} \left( 1 + \frac{2 \|p - q\|^2}{(1 - \|p\|^2)(1 - \|q\|^2)} \right) $$

You don't need to calculate this to use the protocol, but it's fundamental to why the network is so efficient at storing and retrieving contextual information.

Tutorial 1: Sending Your First Packet

Now, let's put it all together. We will construct a simple packet to make an ASSERT (0x002) in the "Tutorial Sandbox" context (0x1A4) with a simple payload.

This is where you will add your long, detailed tutorial content, complete with code blocks, step-by-step instructions, and expected outputs.


from shd_client import Symbol, Context, Operator

# 1. Define our context
# (The client translates 'sandbox' to the correct vector 0x1A4)
ctx = Context.get("tutorial_sandbox") 

# 2. Define our operator
op = Operator.ASSERT

# 3. Define our payload
# (The client handles hashing this data to 24 bits)
payload = {"msg": "hello_world"}

# 4. Construct the 64-bit symbol
symbol_packet = Symbol(context=ctx, operator=op, payload=payload)

# 5. Send the packet to the network
network_response = symbol_packet.send()

print(f"Packet sent (64-bit): {symbol_packet.to_int()}")
print(f"Network response: {network_response.status}")
